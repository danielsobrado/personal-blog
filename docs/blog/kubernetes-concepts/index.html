Kubernetes Concepts

  • Posted By Daniel Sobrado
  • On 10 Oct
  • Category: Programming

In this blog article, we will go over some fundamental Kubernetes ideas. Kubernetes is an extremely capable framework for managing containerized applications in a clustered environment. It intends to improve the handling of connected, distributed components across various infrastructure. These ideas will help you comprehend what Kubernetes is capable of.

Pods

A pod is the basic unit of work in Kubernetes. It is a group of one or more containers that are scheduled together on the same node. Pods share resources such as CPU, memory, and storage.

The reason for using pods rather than single containers is because there may be instances where numerous containers are closely related and it makes sense to operate them on the same machine. Pods are intended to provide co-located (on the same node), co-scheduled (start/stop at the same time) containers, much like a single virtual machine.

Nodes:

A node is a physical or virtual machine that runs Kubernetes. Nodes are responsible for running pods and providing resources to pods.

Nodes are in charge of running pods and supplying resources to pods. They are managed by the control plane and can be thought of as Kubernetes cluster workers.

Cluster:

A cluster is a group of nodes that are managed by Kubernetes. A cluster can be made up of a single node or multiple nodes.

The power of Kubernetes comes from the fact that it can manage clusters of machines and make them act like one big machine.

Control plane:

The control plane is the brains of Kubernetes. It is responsible for scheduling pods, managing resources, and monitoring the cluster.

It includes components like the kube-apiserver, etcd, kube-scheduler, kube-controller-manager, and cloud-controller-manager.

Services:

A service is a logical grouping of pods that are exposed to the outside world. Services provide a single point of access to pods, even if the pods are running on different nodes.

Services are essentially a way to define policies for how traffic should be directed to pods.

Labels:

Labels are used to identify and group pods. Labels can be used to select pods for scheduling, routing, and other operations.

They are a versatile tool for dealing with pod sets and can be used in a variety of ways.

Deployments:

A deployment is a way to manage the lifecycle of pods. Deployments can be used to create, update, and delete pods.

They specify the expected state of pods and can update them automatically using a rolling update technique, assuring pod availability and scalability.

DaemonSets:

A daemonset ensures that a pod is running on every node in the cluster. DaemonSets are often used to run services that need to be available on all nodes, such as logging and monitoring services.

Ingresses:

An ingress is a way to expose services to the outside world. Ingresses can be used to route traffic to different services, balance traffic, and provide SSL/TLS termination.

Advanced Techniques

While the basic Kubernetes concepts introduced above can handle a majority of use-cases, some advanced features further enhance the capabilities of Kubernetes:

  • StatefulSets: While deployments are perfect for stateless applications, StatefulSets provide guarantees about the ordering and uniqueness of Pods, ideal for stateful applications like databases.

  • Jobs and CronJobs: Jobs are used when work needs to be done once, while CronJobs are for tasks that need to be done periodically.

  • ConfigMaps and Secrets: These are used to manage the configuration of your applications separate from your container image, enhancing the portability and security of your applications.

Horizontal Pod Autoscaling: This feature allows Kubernetes to automatically add or remove Pods from a deployment or replica set based on the CPU usage or other select metrics.

Conclusion

Kubernetes has revolutionized the way we build and deploy our applications, providing us with the ability to orchestrate and manage containers at scale. With a thorough understanding of its core concepts, we can design systems that are resilient, scalable, and efficient.
