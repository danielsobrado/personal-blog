Kubernetes Concepts

  • Posted By Daniel Sobrado
  • On 10 Oct
  • Category: Programming

In this blog piece, we’re delving into the world of Kubernetes, an exceedingly proficient system designed to streamline the management of containerized applications in a cluster environment. Our journey will be centered around understanding the essential concepts that underpin Kubernetes and exploring its vast capabilities.

Pods

In the realm of Kubernetes, a Pod represents the primary building block. It’s a cluster of one or more containers programmed to run in unison on a single node. Each pod is allocated a shared pool of resources such as CPU, memory, and storage.

Deploying pods instead of individual containers is a strategic move designed to handle scenarios where various containers are interdependent and need to function on a single machine. Pods are architected to facilitate co-located (on the same node) and co-scheduled (starting/stopping simultaneously) containers, functioning much like a standalone virtual machine.

Nodes

Nodes in Kubernetes are either physical or virtual machines, responsible for the execution of pods and allocation of resources. Acting as the worker bees of the Kubernetes cluster, nodes execute the instructions of the control plane.

Cluster

The Cluster, a collection of nodes supervised by Kubernetes, forms the backbone of the system. It is this cluster mechanism that endows Kubernetes with its superpower – the ability to manage multiple machines as a unified entity.

Control Plane

Deemed the brain of Kubernetes, the Control Plane oversees pod scheduling, resource allocation, and cluster monitoring. It constitutes components like the kube-apiserver, etcd, kube-scheduler, kube-controller-manager, and cloud-controller-manager.

Services

Services are logically assembled pods rendered accessible to external users. By offering a singular point of access to pods operating on different nodes, services shape traffic rules for pod interactions.

Labels

Labels function as identifiers to categorize and group pods, playing a pivotal role in pod selection for tasks such as scheduling, routing, and other operations.

Deployments

Deployment stands as the mechanism to regulate the life cycle of pods. It can help to create, update, and delete pods. These deployments define the anticipated state of pods and facilitate automatic updates through a rolling update technique, ensuring pod availability and scalability.

DaemonSets

DaemonSets serve to guarantee the execution of a pod on each node within the cluster. This feature is particularly advantageous for services such as logging and monitoring, which need to be omnipresent across all nodes.

Ingresses

An Ingress is a pathway to expose services to external users. It is primarily leveraged to direct traffic to distinct services, distribute traffic, and provide SSL/TLS termination.

Advanced Techniques

Apart from the primary Kubernetes concepts delineated above, there are advanced features that further augment the potential of Kubernetes:

  • StatefulSets: Ideally suited for stateful applications like databases, StatefulSets assure the order and uniqueness of Pods. This is in contrast to Deployments, which are tailored for stateless applications.

  • Jobs and CronJobs: While Jobs are employed for tasks that need a one-time execution, CronJobs cater to tasks that necessitate periodic execution.

  • ConfigMaps and Secrets: These mechanisms manage your applications’ configuration independent of your container image, thereby enhancing portability and security.

  • Horizontal Pod Autoscaling: This functionality enables Kubernetes to automatically scale the number of Pods in a Deployment or Replica Set based on CPU utilization or other specified metrics.

Wrapping Up

Kubernetes has revolutionized the way we develop and deploy applications. It equips us with the tools to orchestrate and manage containers on a massive scale. With a comprehensive grasp of its core concepts, we can design robust, scalable, and efficient systems.
